# 회의실 예약 앱 — 기능 단위 백엔드 개발 플랜 (Postman 검증)

## Context

엔터티 레이어가 완성된 상태에서, **기능 하나씩 끝까지 완성하면서 Postman으로 즉시 검증**하는 방식으로 개발한다.
레이어 단위(Repository 전체 → DTO 전체 → ...)가 아닌, 수직 슬라이스(Vertical Slice)로 진행한다.

---

## 개발 흐름

```
Phase 0: 공통 기반 (Exception, 빈 Repository)
   ↓
Phase 0.5: 스터디룸 초기 데이터 삽입 (CommandLineRunner)
   ↓
Phase 1: 회원가입 → Postman 검증
   ↓
Phase 2: 로그인 + 로그아웃 → Postman 검증
   ↓
Phase 3: 내 정보 조회 → Postman 검증
   ↓
Phase 4: 회의실 CRUD → Postman 검증
   ↓
Phase 5: 예약 생성 → Postman 검증
   ↓
Phase 6: 내 예약 조회 → Postman 검증
   ↓
Phase 7: 예약 취소 → Postman 검증
   ↓
Phase 8: 날짜별 예약 조회 → Postman 검증
```

### exception/GlobalExceptionHandler.java

```java
package com.pknu26.roomy.exception;

import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(CustomException.class)
    public ResponseEntity<ErrorResponse> handleCustomException(CustomException e) {
        ErrorCode errorCode = e.getErrorCode();
        return ResponseEntity
                .status(errorCode.getStatus())
                .body(ErrorResponse.of(errorCode));
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ErrorResponse> handleValidException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult()
                .getFieldErrors()
                .get(0)
                .getDefaultMessage();
        return ResponseEntity
                .status(HttpStatus.BAD_REQUEST)
                .body(new ErrorResponse(400, message));
    }
}
```

```java
        FROM Reservation r

        AND r.reservationDate = :date

        """)

                          @Param("startTime") LocalTime startTime,
                          @Param("endTime") LocalTime endTime);
```

```java
package com.pknu26.roomy.init;

import com.pknu26.roomy.entity.MeetingRoom;

import org.springframework.boot.CommandLineRunner;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import java.util.List;

@Configuration
public class MeetingRoomInitializer {

                    MeetingRoom.builder().name("스터디룸 D").capacity(6).minHeadcount(4).build(),
                    MeetingRoom.builder().name("스터디룸 E").capacity(6).minHeadcount(4).build(),
                    MeetingRoom.builder().name("스터디룸 F").capacity(6).minHeadcount(4).build(),
                    MeetingRoom.builder().name("스터디룸 G").capacity(8).minHeadcount(4).build(),
```

### service/UserService.java (초기 버전)

```java
package com.pknu26.roomy.service;

import com.pknu26.roomy.dto.request.SignUpRequest;
import com.pknu26.roomy.dto.response.UserResponse;
import com.pknu26.roomy.entity.User;
import com.pknu26.roomy.exception.CustomException;
import com.pknu26.roomy.exception.ErrorCode;
import com.pknu26.roomy.repository.UserRepository;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
@RequiredArgsConstructor
@Transactional
public class UserService {

    private final UserRepository userRepository;

    public UserResponse signUp(SignUpRequest request) {
        if (userRepository.existsByUsername(request.getUsername())) {
            throw new CustomException(ErrorCode.USERNAME_DUPLICATED);
        }
        User user = User.builder()
                .username(request.getUsername())
                .password(request.getPassword())
                .build();
        return UserResponse.from(userRepository.save(user));
    }
}
```

### controller/UserController.java (초기 버전)

```java
package com.pknu26.roomy.controller;

import com.pknu26.roomy.dto.request.SignUpRequest;
import com.pknu26.roomy.dto.response.UserResponse;
import com.pknu26.roomy.service.UserService;
import jakarta.validation.Valid;
import lombok.RequiredArgsConstructor;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/users")
@RequiredArgsConstructor
public class UserController {

    private final UserService userService;

    @PostMapping("/signup")
    public ResponseEntity<UserResponse> signUp(@RequestBody @Valid SignUpRequest request) {
        UserResponse response = userService.signUp(request);
        return ResponseEntity.status(HttpStatus.CREATED).body(response);
    }
}
```

> **목표 API**: `POST /api/users/login`, `POST /api/users/logout`
> 세션 기반 인증이 처음 등장한다. Postman의 쿠키 동작 이해가 중요.

### service/UserService.java (메서드 추가)

```java
// 클래스 내부에 메서드 추가
@Transactional(readOnly = true)
public User login(LoginRequest request) {
    User user = userRepository.findByUsername(request.getUsername())
            .orElseThrow(() -> new CustomException(ErrorCode.USER_NOT_FOUND));
    if (!user.getPassword().equals(request.getPassword())) {
        throw new CustomException(ErrorCode.INVALID_PASSWORD);
    }
    return user;
}
```

### controller/UserController.java (메서드 추가)

```java
// 클래스 최상단에 상수 추가
private static final String SESSION_USER_ID = "LOGIN_USER_ID";

// 메서드 추가
@PostMapping("/login")
public ResponseEntity<UserResponse> login(@RequestBody @Valid LoginRequest request,
                                           HttpSession session) {
    User user = userService.login(request);
    session.setAttribute(SESSION_USER_ID, user.getId());
    return ResponseEntity.ok(UserResponse.from(user));
}

@PostMapping("/logout")
public ResponseEntity<Void> logout(HttpSession session) {
    session.invalidate();
    return ResponseEntity.ok().build();
}
```

>
> - `import com.pknu26.roomy.dto.request.LoginRequest;`
> - `import com.pknu26.roomy.entity.User;`

## 🧪 Phase 2 Postman 테스트

### 1) 정상 로그인

```
POST {{BASE_URL}}/api/users/login
Headers: Content-Type: application/json
Body:
{
  "username": "testuser",
  "password": "password123"
}
```

**기대 응답**: `200 OK`

```json
{ "id": 1, "username": "testuser" }
```

**중요 확인사항**:

- Postman 하단 **Cookies 탭** → `JSESSIONID`가 생성됐는지 확인
- 이 쿠키가 이후 로그인 필요 API의 인증 수단이 됨

### 2) 존재하지 않는 아이디

```json
{ "username": "notexist", "password": "password123" }
```

**기대 응답**: `404 Not Found`

### 3) 비밀번호 틀림

```json
{ "username": "testuser", "password": "wrongpass" }
```

**기대 응답**: `401 Unauthorized`

### 4) 로그아웃

```
POST {{BASE_URL}}/api/users/logout
```

**기대 응답**: `200 OK`

- 로그아웃 후 Cookies 탭에서 `JSESSIONID`가 제거되거나 무효화됨

```java
@GetMapping("/me")
public ResponseEntity<UserResponse> getMyInfo(HttpSession session) {
    Long userId = getLoginUserId(session);
    User user = userService.findById(userId);
    return ResponseEntity.ok(UserResponse.from(user));

// 헬퍼 메서드 (클래스 마지막에 추가)
private Long getLoginUserId(HttpSession session) {

        throw new CustomException(ErrorCode.UNAUTHORIZED);

    return userId;
```

### service/ReservationService.java

```java
package com.pknu26.roomy.service;

import java.time.Duration;
import java.time.LocalTime;

import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import com.pknu26.roomy.dto.request.ReservationCreateRequest;
import com.pknu26.roomy.dto.response.ReservationResponse;
import com.pknu26.roomy.entity.MeetingRoom;
import com.pknu26.roomy.entity.Reservation;
import com.pknu26.roomy.entity.User;
import com.pknu26.roomy.exception.CustomException;
import com.pknu26.roomy.exception.ErrorCode;
import com.pknu26.roomy.repository.MeetingRoomRepository;
import com.pknu26.roomy.repository.ReservationRepository;
import com.pknu26.roomy.repository.UserRepository;
import lombok.RequiredArgsConstructor;

@Service
@RequiredArgsConstructor
@Transactional
public class ReservationService {

    private final ReservationRepository reservationRepository;
    private final UserRepository userRepository;
    private final MeetingRoomRepository meetingRoomRepository;

    public ReservationResponse create(Long userId, ReservationCreateRequest request) {
        User user = userRepository.findById(userId)
                .orElseThrow(() -> new CustomException(ErrorCode.USER_NOT_FOUND));
        MeetingRoom room = meetingRoomRepository.findById(request.getRoomId())
                .orElseThrow(() -> new CustomException(ErrorCode.ROOM_NOT_FOUND));

        validateReservationTime(request.getStartTime(), request.getEndTime());

        if (request.getHeadCount() < room.getMinHeadcount()) {
            throw new CustomException(ErrorCode.RESERVATION_BELOW_MIN_CAPACITY);
        }

        if (reservationRepository.existsOverlap(
                room, request.getReservationDate(),
                request.getStartTime(), request.getEndTime())) {
            throw new CustomException(ErrorCode.RESERVATION_DUPLICATED);
        }

        Reservation reservation = Reservation.builder()
                .user(user)
                .room(room)
                .reservationDate(request.getReservationDate())
                .startTime(request.getStartTime())
                .endTime(request.getEndTime())
                .headCount(request.getHeadCount())
                .build();

        return ReservationResponse.from(reservationRepository.save(reservation));
    }

    // LocalTime은 Java의 java.time 패키지에 있는 시간만 표현하는 클래스
    private void validateReservationTime(LocalTime start, LocalTime end) {
        if (!start.isBefore(end)) {
            throw new CustomException(ErrorCode.INVALID_RESERVATION_TIME);
        }
        long minutes = Duration.between(start, end).toMinutes();
        if (minutes < 30 || minutes > 240) {
            throw new CustomException(ErrorCode.INVALID_RESERVATION_DURATION);
        }
    }
}
```

# 트러블슈팅

## Postman에서 401만 계속 떨어질 때

- Postman 하단 Cookies 탭에서 `JSESSIONID` 존재 여부 확인
- 없으면 로그인 다시 → 같은 Postman 창에서 후속 요청

## 500 에러가 떴을 때

- IntelliJ 콘솔 로그의 스택 트레이스 먼저 확인
- 보통 NullPointerException, JPA LazyInitializationException 등

## DB에 데이터가 안 보일 때

- `application.properties`의 `ddl-auto` 값 확인
  - `create` : 앱 시작 시 테이블 초기화 (데이터 사라짐)
  - `update` : 스키마만 변경, 데이터 유지
- sqlplus에서 `commit`이 안 됐을 수도 있음 (JPA는 자동 commit)

## `@PathVariable` 파라미터 이름 오류 (단건 조회 500 에러)

**증상**: `GET /api/rooms/{id}` 요청 시 아래 에러 발생
```
IllegalArgumentException: Name for argument of type [java.lang.Long] not specified,
and parameter name information not available via reflection.
Ensure that the compiler uses the '-parameters' flag.
```

**원인**: Java는 기본적으로 컴파일된 바이트코드에 파라미터 이름을 포함하지 않는다.
`@PathVariable Long id`처럼 이름을 생략하면 Spring이 리플렉션으로 이름을 추론하려 하지만,
`-parameters` 컴파일러 플래그가 없으면 추론에 실패하여 에러가 발생한다.

**해결**: `@PathVariable`에 이름을 명시한다.
```java
// 변경 전 (문제)
public ResponseEntity<MeetingRoomResponse> findById(@PathVariable Long id)

// 변경 후 (정상)
public ResponseEntity<MeetingRoomResponse> findById(@PathVariable("id") Long id)
```

> `@RequestParam`을 쓸 때도 동일한 문제가 생길 수 있으므로, 이름을 항상 명시하는 것이 안전하다.
